# Accessibility

The portal must be keyboard-friendly and screen-reader friendly by default.

## Forms

- Visible labels for every field.
- `aria-describedby` for hints and errors.
- `aria-invalid` for invalid fields.
- Focus first invalid field after validation failure.
- Focus step heading after wizard step changes.

## UI

- Real buttons for actions.
- Real links for navigation.
- Semantic tables for candidate data.
- Visible focus states.
- Status conveyed with text, not color alone.
